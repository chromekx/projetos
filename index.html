Git e Github

O que é o Git?

O Git é um sistema de controle de versão distribuído. Registra históricos de alterações em arquivos (normalmente código), permite criar branches (linhas de desenvolvimento) e voltar a estados anteriores. Foi feito para rastrear mudanças e evitar que equipes destruam trabalho umas das outras por acidente.

O que é o GitHub?

O GitHub é uma plataforma online que hospeda repositórios Git. Oferece interface web, issues, pull requests, integração contínua e ferramentas de colaboração. GitHub não substitui o Git: ele usa Git como tecnologia subjacente, fornece hospedagem e funcionalidades sociais.

Quais são suas diferenças?

1 - Git: ferramenta local (linha de comando ou GUI), controla versão no seu computador.
2 - GitHub: serviço remoto (web) que armazena repositórios Git e adiciona recursos sociais/colaborativos.
3 - Você pode usar Git sem GitHub; não dá pra usar GitHub sem Git.

Comandos

Principais Comandos do GitBash

## Definir seu nome
git config --global user.name "Seu nome"

## Definir seu email
git config --global user.email seu@email

## Iniciar o rastreamento do git na pasta atual
git init

## Pedir dados do rastreamento do git, como arquivos não rastreados
git status

## Adicionar um arquivo ao rastreamento do git
git add arquivo

## Verificar o log das atualizações
git log

## Gerar uma versão do projeto
git commit -m "explicação"

Repositórios

Como eu crio um Repositório no GitHub?

1 - Faça login no site do GitHub
2 - Clique no botão "New repository".
3 - Preencha o nome do repositório, descrição, e escolha se ele será público ou privado.

Como eu ligo um projeto existente no GitHub no meu PC?

git commit -m "commit"
# adicionar o repositório remoto -
-> substitua o link pelo que foi copiado do GitHub
git remote add origin (link copiado)
git push -u origin main

Como eu gero uma versão do projeto com o Git?

# criar um commit
git commit -m "Descrição clara da mudança"
# criar uma tag anotada para marcar uma versão
git tag -a v1.0.0 -m "Versão 1.0.0"
# enviar tags para o remoto
git push origin --tags

Como eu envio do meu PC para o Github?

# garantir que você tem rep. remoto configurado
git remote -v
# enviar a branch atual para o remoto -
-> só precisa do -u na primeira vez
git push -u origin main
# nas próximas vezes, só
git push
